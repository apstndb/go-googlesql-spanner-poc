# Compact-Tree-Metadata Plan Shape Observations

Observed on 2026-05-07 with Spanner Omni through `spanemuboost`. This file records the regenerated `--output compact-tree-metadata` results for the verification cases referenced by the query-plan-shape research notes.

Each result preserves visible child-link topology with edge labels such as `-[Build]->`, `-[Probe]->`, `-[Input]->`, and `-[Map]->`, while omitting PlanNode indexes by default. A child link is visible when the child PlanNode kind is `RELATIONAL` or the link type is `Scalar`, matching `spannerplan.QueryPlan.IsVisible`. Hidden scalar links are grouped by child PlanNode `DisplayName`, such as `Function[Residual Condition]`, `Reference[Output]`, or `Array Constructor`. Links already rendered as tree edges are omitted from the annotation.

The broader 2040-entry `statement_hint_query_matrix` is intentionally kept as a success/error summary in [`QUERY_EXECUTION_OPERATORS_OBSERVATIONS.md`](QUERY_EXECUTION_OPERATORS_OBSERVATIONS.md#broad-statement-hint-query-matrix); expand it with `--output compact-tree-metadata` only when a specific hint/query combination needs inspection.
